# CLAUDE.md — uzcy

Reverse include dependency tracker for C/C++. Scans a tree, builds a forward
`#include` map, inverts it, and writes `// <used_by>` comment blocks into each
included file.

## Layout

`src/` layout, hatchling + hatch-vcs (version from git tags).

- `this_file: src/uzcy/cli.py` — Fire CLI, config assembly, backend selection, run loop.
- `this_file: src/uzcy/config.py` — `UzcyConfig` (pydantic), default excludes, C/C++ extensions.
- `this_file: src/uzcy/discovery.py` — recursive file discovery with exclude filtering.
- `this_file: src/uzcy/clang_backend.py` — `clang-scan-deps` driver + tolerant JSON parsing.
- `this_file: src/uzcy/text_scan.py` — regex `#include` scanner fallback (comment-aware).
- `this_file: src/uzcy/maps.py` — forward map normalization and inversion to `used_by`.
- `this_file: src/uzcy/writer.py` — idempotent, atomic block insert/replace/remove.
- `this_file: src/uzcy/paths.py` — path resolution and POSIX-relative formatting.

## Architecture notes

- Two backends, one contract: both produce a forward map `{file -> [includes]}`
  restricted to files known in the tree. `maps.invert_forward_map` turns that
  into `{file -> [users]}`.
- `auto` mode prefers clang (accurate, needs `compile_commands.json`) and falls
  back to the text scanner; a clang failure logs and degrades, never aborts.
- The writer is the delicate part: it detects and preserves newline style and
  trailing newline, finds the block by its `// <used_by>` / `// </used_by>`
  markers, and writes atomically via a temp file + replace.

uv run ruff check . && uv run ruff format --check .

## Gotchas

- No git tag yet means hatch-vcs emits a `0.1.devN` version. `v1.0.0` is the
  baseline release.
- `src/uzcy/__version__.py` is generated by hatch-vcs — gitignored, never edited.
- Not related to Cython. "cy" = C/C++, the languages it scans; it is a Python tool.
